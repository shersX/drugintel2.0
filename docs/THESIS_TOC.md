#### 1.1 研究背景与意义

##### 1.1.1 医药竞争情报的业务痛点与需求

**写作要点：**

#### 4.2 基于向量相似度的增量式事件聚类算法

> **本节是论文最核心的创新点，需要重点展开。**

##### 4.2.1 多阈值链式聚类算法设计

**写作要点：**
- 问题定义：将 N 条医药新闻按事件归属聚类，同一事件的多篇报道应归入同一簇
- 与经典聚类算法的区别：不需要预设簇数 K、支持增量更新、阈值可解释
- 算法流程（伪代码）：
  1. 初始化：所有新闻标记为"未分配"
  2. 取未分配集合中第一条新闻作为种子，创建新事件簇
  3. **高阈值扩展**：对未分配新闻，若与种子的余弦相似度 >= θ_high，直接加入簇
  4. 计算当前簇的中心向量（L2 归一化后的均值）
  5. **中阈值扩展**：对未分配新闻，若与种子的相似度 >= θ_mid 且与中心向量的相似度 >= θ_mid，加入簇
  6. 簇闭合，回到步骤 2 处理下一个种子
- 配算法流程图和伪代码框
- 时间复杂度分析：O(N^2) 最坏情况，实际因簇内规模有限远低于此

##### 4.2.2 事件中心向量动态更新策略

**写作要点：**
- 增量场景：新新闻到达后，需要判断是否属于已有事件
- 中心向量更新公式：c_new = (1-α) * c_old + α * v_new，然后 L2 归一化
- α 的选择与意义（新闻权重 vs 历史权重）
- 增量更新避免了全量重聚类的开销

##### 4.2.3 代表新闻自动选择算法

**写作要点：**
- 代表新闻的定义：每个事件簇中最能概括该事件的一条新闻
- 选择策略：选择与簇中心向量余弦相似度最高的新闻
- 平局处理：若多条新闻相似度相同，取发布时间最新的
- 代表新闻用于前端展示和告警推送

##### 4.2.4 与传统聚类算法的对比分析

**写作要点：**

**实验设计：**

**对比方法：**

## 创新点总结

> 建议写在摘要和 1.3 节中。

1. **提出了一种多层级医药新闻过滤机制**，将关键词初筛与大模型语义相关性判断相结合，在控制 LLM 调用成本的前提下有效提升了医药新闻的过滤准确率。

2. **设计了基于向量相似度的增量式事件聚类算法**，采用多阈值链式聚类与中心向量动态更新策略，无需预设簇数即可实现对医药事件的自动归并与跟踪；实验表明，该方法在 ARI 和成对 F1 指标上优于标题去重、单阈值聚类、K-Means 和 DBSCAN 等基线方法。

3. **构建了面向医药领域的 RAG 检索增强问答系统**，融合向量检索与全文检索（RRF 融合），结合 Cross-Encoder 重排序和领域意图识别，实验表明混合检索+重排序策略在 P@10 和 MRR 指标上显著优于单一检索方式。
