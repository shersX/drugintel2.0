# 论文「一类创新」：事件级聚类与评测 —— 从零开始的实施步骤（仅文档）

本文档面向**代码在其它仓库、本仓库视为空项目**的情况，说明如何**仅通过工程与实验设计**落地「医药多源新闻 → 事件级聚合」这一类方法创新，并产出**可写进硕士论文的表格与论述**。**不包含代码**，只精确到步骤与交付物格式。

---

## 0. 目标与边界

### 0.1 论文要宣称的一句话（实验完成后可写）

在统一文本向量空间下，针对医药多源新闻的**同题重复**，给出**分层阈值 + 簇中心约束的链式聚类**流程；与**标题去重**、**单阈值向量聚类**等基线相比，在自建标注集上**聚类一致性指标（如 ARI）或成对 F1** 更优，并辅以**消融实验**说明各组件的作用。

### 0.2 不宣称的内容

- 不宣称发明新的基础模型或 Embedding 架构。
- 不宣称「向量聚类」本身为新理论；创新在于**领域问题上的流程设计 + 可复现对比 + 指标**。

### 0.3 最终交付物清单（答辩与论文用）

| 序号 | 交付物 | 用途 |
|------|--------|------|
| D1 | 标注规范说明（1～2 页） | 方法可信度 |
| D2 | 金标准数据文件（格式见下文） | 复现 |
| D3 | 各基线与本文方法的**预测簇 ID**（同格式） | 算指标 |
| D4 | 验证集上调参记录（表格或 YAML 清单） | 避免测试集泄漏 |
| D5 | 测试集主结果表 + 消融表 | 论文第 6/7/11 章 |
| D6 | 随机种子与软件版本记录 | 可复现 |

## 1. 阶段划分与时间顺序

按**严格顺序**执行，不可跳步（跳步会导致「测试集调参」等硬伤）。

1. **阶段 A**：固定向量定义与数据导出规范。  
2. **阶段 B**：设计标注规范与抽样方案。  
3. **阶段 C**：完成金标准标注（先小样本试标，再全量）。  
4. **阶段 D**：划分训练 / 验证 / 测试（按**事件**或按**新闻**策略二选一，见 §5）。  
5. **阶段 E**：实现基线 B1、B2（逻辑在任意仓库实现，本文只规定输入输出）。  
6. **阶段 F**：在**验证集**上网格搜索本文方法与 B2 的超参数。  
7. **阶段 G**：在**测试集**上跑一次，生成主表。  
8. **阶段 H**：消融实验（在验证集选配置，测试集报告一次）。  
9. **阶段 I**：撰写论文中的「方法」「实验」「消融」三节，与 D1～D6 对齐。

## 2. 阶段 A：向量与数据字段（必须先冻结）

### 步骤 A1：选定 Embedding 模型

- 选定**唯一**模型（例如 PRD 中的 BGE-M3 或你实际使用的模型），记录：**模型名、向量维度、是否 L2 归一化**。  
- **全文实验禁止更换模型**，否则对比无效。

### 步骤 A2：规定每条样本的必备字段

每条新闻（或文档）在实验数据集中至少包含：

| 字段名 | 含义 | 说明 |
|--------|------|------|
| `sample_id` | 稳定唯一 ID | 整型或 UUID 字符串，全局唯一 |
| `title` | 标题 | 用于 B1 基线 |
| `text_for_embed` | 用于算向量的文本 | 建议固定规则：如 `title + "\n" + abstract` 或截断正文前 N 字 |
| `embedding` | 向量 | 长度 = 维度；与线上一致 |
| `publish_time` | 发布时间 | 可选；用于代表稿 tie-break |
| `source` | 来源站点 | 可选；分层抽样用 |

### 步骤 A3：导出「实验子集」

- 从业务库中导出 **N 条**记录（建议 **N ≥ 400**，若资源有限不少于 **250**），保存为**中间格式**（如 JSON Lines / Parquet / CSV+单独向量文件）。  
- 要求：**embedding 已算好**再导出，避免标注后补算导致与线上一致性争议。

### 步骤 A4：冻结「实验数据集版本」

- 为本次导出命名版本号，如 `dataset_v1_20260406`。  
- 论文中所有表格注明：**基于 `dataset_v1_*`**。

---

## 3. 阶段 B：标注规范（D1 文档内容）

### 步骤 B1：定义「事件」粒度（必须写下来）

在医药新闻场景，规定**何种情况算同一事件**，例如（按你学科方向微调）：

- **同一事件**：围绕**同一监管动作 / 同一临床试验结果 / 同一并购公告**的多篇通稿、转载、简讯。  
- **不同事件**：同一药物但**不同阶段**（获批 vs III 期失败）视为不同事件。  
- **边界案例**：写 2～3 个**正例**、**反例**文字说明，减少标注者漂移。

### 步骤 B2：分配 `event_gold_id`

- 为每条 `sample_id` 赋予一个整数 `event_gold_id`。  
- **同一事件内**所有新闻共享同一 `event_gold_id`。  
- **不需要**与业务库的 `event_id` 一致；金标准独立编号即可。

### 步骤 B3：试标（Pilot）

- 随机抽 **30～50 条**，由**主标注人**标完。  
- 再抽 **30 条**由**第二人**独立标注，计算**事件级一致性**（或成对是否同事件的一致性）。  
- 若一致性低：回到 B1 细化规则，**不要**直接扩大量。

### 步骤 B4：正式标注

- 主标注人完成全量；或双人交叉 + 第三人仲裁（时间够再用）。  
- 记录标注日期与标注人角色（论文附录一句话）。

---

## 4. 金标准文件格式（D2）

建议统一为 **JSON Lines**，每行一个对象，便于脚本处理：

```text
{"sample_id": "...", "event_gold_id": 12, "title": "...", "meta": {"source": "..."}}
```

向量若体积大，可**分文件**：`embeddings/{sample_id}.npy` 或单一 `embeddings.npz` 与 `id_order.txt` 对齐。

**冻结检查**：每个 `sample_id` 恰好一行；`event_gold_id` 无空值。

---

## 5. 阶段 D：数据集划分策略

### 策略一（推荐）：按事件划分

- 列出所有 distinct `event_gold_id`。  
- 将 **event_id** 随机打乱，按 **70% / 15% / 15%** 划到 train / val / test。  
- 把所有属于测试集事件的样本划入测试集，其余类推。  
- **优点**：避免同一事件泄漏到训练与测试，指标更严。

### 策略二：按样本随机划分（仅事件很少时慎用）

- 简单随机分样本；若同一事件被切开，会**乐观偏误**，论文中需说明局限性。

**输出**：三个列表文件 `train_ids.txt`、`val_ids.txt`、`test_ids.txt`（每行一个 `sample_id`）。

---

## 6. 相似度与数学约定（所有方法共用）

### 步骤 M1：余弦相似度

- 若向量已 L2 归一化：两向量点积即余弦相似度。  
- 记录公式于论文：\( \cos(\mathbf{a},\mathbf{b}) = \mathbf{a}^\top \mathbf{b} \)（归一化后）。

### 步骤 M2：簇中心

- 给定簇内若干向量，中心为**算术平均**，再 **L2 归一化**（若后续用余弦相似度）。

---

## 7. 基线 B1：标题去重（实现逻辑的步骤级说明）

### 步骤 B1-1：标题规范化函数（口头算法）

对每条标题依次：

1. 去除首尾空白；全半角统一；连续空白压成单空格。  
2. （可选）转小写（英文稿多时有帮助）。  
3. （可选）去掉「转载」「快讯」等后缀列表（需列在附录，避免随意）。

### 步骤 B1-2：聚类规则

- 规范化后 **字符串完全相等** 的样本归为同一簇。  
- 每个连通组分配一个递增的 `cluster_id`。

### 步骤 B1-3：输出

- 文件：`pred_b1.jsonl`，每行 `sample_id` + `cluster_pred_id`。

**论文表述**：B1 **不使用**向量，代表传统编辑侧去重。

---

## 8. 基线 B2：单阈值向量聚类

### 步骤 B2-1：输入

- 仅使用某子集（train+val 或全数据用于最终测试前，**严格按 §12 执行**）：样本与 `embedding`。

### 步骤 B2-2：算法（贪心链式，与 PRD  spirit 接近但只一阈值）

**过程描述**：

1. 维护「未分配」集合，初始为全部样本。  
2. 当未分配非空：从未分配中取一条作为**种子**，开新簇，编号新 `cluster_id`。  
3. 扫描其余未分配样本，若 `cos(seed, x) ≥ τ`，则加入本簇并从「未分配」移除。  
4. 种子与已入簇样本**不再**作为新种子（除非你在附录采用另一种顺序策略，需固定并记录）。  
5. 重复 2～4 直到未分配为空。

### 步骤 B2-3：超参数 τ

- **仅在验证集**上从候选集合选最优，例如：`{0.70, 0.75, 0.80, 0.85, 0.90, 0.95}`。  
- **禁止**用测试集选 τ。

### 步骤 B2-4：输出

- `pred_b2_tau085.jsonl`（文件名带最终选定 τ，论文写清）。

## 9. 本文方法：分层阈值 + 簇中心（与 PRD 对齐的实现步骤）

下列步骤须在**同一套**样本顺序与随机种子下实现，便于复现。

### 步骤 P1：初始化

- 未分配集合 = 全部样本；簇列表为空。

### 步骤 P2：选种子

- 从未分配中取第一条（或按 `publish_time` 最早，**二选一写死**）作为种子，建新簇。

### 步骤 P3：第一轮扩展（高阈值）

- 对未分配中每条 `x`，若 `cos(seed, x) ≥ θ_high`，则加入簇并移出未分配。

### 步骤 P4：计算当前簇中心

- 对簇内所有 `embedding` 求平均，再 L2 归一化，记为 `c`。

### 步骤 P5：第二轮扩展（中阈值 + 中心）

- 对仍「未分配」的 `x`，若 `cos(seed, x) ≥ θ_mid` **且** `cos(c, x) ≥ θ_mid`，则加入簇并移出未分配。  
- （若 PRD 中还有低阈值分支：）对满足 `θ_low ≤ cos(seed, x) < θ_mid` 的样本，再判断与 `c` 是否 ≥ 某门槛；**具体分支必须与 PRD 定稿一致**，此处只保留占位，你实现时填实数。

### 步骤 P6：迭代

- 若簇在 P4～P5 中有新成员，**可选**：重新计算 `c` 并再扫一轮未分配（**轮数上限**写死为 2 或 3，防止死循环）。  
- 记录该设计于论文「算法说明」。

### 步骤 P7：簇闭合

- 当前簇不再增长后，从「未分配」取下一个种子，重复 P2～P6。

### 步骤 P8：代表稿（若论文需要业务指标）

- 对每个簇，计算与 `c` 余弦相似度最大的样本为 `representative_id`；若并列，取 `publish_time` 最新。

### 步骤 P9：超参数搜索空间（仅验证集）

- 例如：`θ_high ∈ {0.88, 0.90, 0.92}`，`θ_mid ∈ {0.75, 0.80, 0.85}`，`θ_low` 若不用则关闭。  
- **网格**组合在验证集上算指标，选**一组**最优组合。

### 步骤 P10：输出

- `pred_proposed.jsonl` + 附录记录最终 `(θ_high, θ_mid, …)`。

## 10. 指标计算步骤（D5 的算法说明）

### 10.1 聚类指标：ARI（推荐主指标之一）

**前提**：金标准 `event_gold_id` 视为真实簇标签；预测 `cluster_pred_id` 为算法簇标签。

**步骤**：

1. 将 `event_gold_id`、`cluster_pred_id` 各自重映射为 **0…K-1** 的连续整数标签（顺序无关）。  
2. 构造**列联表** \(n_{ij}\)（真实类 i，预测类 j）。  
3. 按 sklearn 或教科书公式计算 **Adjusted Rand Index**。  
4. 取值范围理论上 ≤1，越大越好。

**报告**：在 **test** 上分别报告 B1、B2、本文方法的 ARI。

### 10.2 成对指标：Precision / Recall / F1

**步骤**：

1. 在某一数据子集（建议 test）上，枚举所有**无序样本对** \((i,j), i<j\)（或对大数据随机抽 **10^5** 对）。  
2. 定义金标准：`same_gold = (event_gold_id[i] == event_gold_id[j])`。  
3. 定义预测：`same_pred = (cluster_pred_id[i] == cluster_pred_id[j])`。  
4. 统计 TP/FP/FN：  
   - TP：`same_gold` 且 `same_pred`  
   - FP：非 `same_gold` 但 `same_pred`  
   - FN：`same_gold` 但非 `same_pred`  
5. 计算 P、R、F1。  

**报告**：与 ARI 二选一为主，另一个放附录或并排。

### 10.3 业务向：列表冗余率（可选）

**步骤**：

1. 对每个真实事件，若算法簇把该事件拆成多个簇，记为「欠合并」；若把多事件并为一簇，记为「过合并」。  
2. 定义简单指标：例如「测试集事件中，被完整合并正确的比例」——需你给出可操作定义，与导师对齐后写入论文。

## 11. 阶段 H：消融实验（最小集合）

在**验证集**选定本文方法最优超参后，固定测试集，仅改下列之一：

| 消融编号 | 改动 | 目的 |
|----------|------|------|
| A1 | 去掉「中心向量」判断，只保留单阈值扩展（退化为变体 B2） | 证明中心项不是冗余 |
| A2 | `θ_high` 与 `θ_mid` 合并为单阈值 | 证明分层有效 |
| A3 | 代表稿策略改为「标题最短」等 | 可选，偏产品 |

每个消融在 **test** 上报一次 ARI（或 F1），做成小表。

---

## 12. 测试集使用纪律（避免学术硬伤）

1. **测试集 `test_ids` 在标注完成后一次性划分，之后不再改动**。  
2. **所有阈值、网格搜索，只允许在 `val` 上选**。  
3. **测试集指标**在论文主表只出现**一次最终配置**；若曾偷看测试集调参，整章无效。  
4. 在论文「实验设置」中明确写：**We tune hyperparameters on the validation set only.**
